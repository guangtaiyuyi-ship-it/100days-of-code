# 100 Days of Code

PythonでのWeb開発の基本を身につけるため、Flaskを使用したWebアプリケーション開発の記録です。

---

## 📅 開発ログ

### 14日目 (7月24日)
- Flask基本操作の復習を実施
- **所感**: 一度実装してから基本を復習することで、前回はイメージできなかった部分もクリアに理解できるようになった。また、これまで使っていなかった機能にも新たに気づくことができた。
 
### 13日目 (7月23日)
- `feature/my-library`（書籍管理アプリ）のブランチを `main` ブランチへマージ完了
- GitHubリポジトリの整理（不要ファイルの削除など）および `README.md` へ1日目から13日目までの開発状況を反映
- **所感**: 一連の機能実装からマージ、ドキュメント整備までの一連の流れを実践できた。

### 12日目 (7月22日)
- ルート関数に共通していたログインチェックを `functools.wraps` を使ってリファクタリング（重複解消）
- 各HTMLに重複していた `<html>` や `<head>` などを `base.html` の継承によって共通化
- **所感**: HTMLのリファクタリングは初めて行ったので学びが多かった。

### 11日目 (7月21日)
- キーワード検索機能を実装
- **所感**: `request.form.get` と `request.args.get` の違いを検索機能の実装を通して理解することができた。

### 10日目 (7月18日)
- `add_book.html`（書籍追加専用ページ）を追加
- 書籍リストをすっきりさせるため、書籍追加を別ページで行うようUIを変更
- **所感**: 画面の役割を明確に分けることで操作性が向上した。

### 9日目 (7月17日)
- 読書状態（読了・未読）の表示を実装
- 書籍の表示形式をリストから一覧表（テーブル）に変更
- **所感**: HTMLのタグは豊富で様々な表現ができると実感。すべてのタグを覚えるのではなく、主要な機能を理解し、具体的なコーディングはAIを活用していく方針に。

### 8日目 (7月16日)
- 書籍リスト用にトップページ、編集画面、状態（完了・未完了）を修正
- **所感**: HTML側を修正する時間が大半だったため、HTMLの書き方がなんとなくわかってきた。

### 7日目 (7月15日)
- mainからブランチし、書籍リストの作成を開始
- **所感**: 基本が理解できているかの確認と応用目的でブランチを実施。

### 6日目 (7月14日)
- ログイン・ログアウト機能を実装
- Webデザインの統一
- **所感**: パスワードのハッシュ化によるセキュリティ向上について、ライブラリを活用して簡単に実装できることを学んだ。

### 5日目 (7月12日)
- タスクの完了と、完了したタスクの一括削除機能を実装
- `.gitignore` の追加
- **所感**: 最初に作るべきだった `.gitignore` を今更ながら追加。一括削除には確認画面の表示を追加するのも検討余地あり。

### 4日目 (7月11日)
- ORMを実装し、SQL言語を省略化
- CSSによるデザイン適用
- **所感**: SQLを実装した後にORMを追加したため、DBの構成がごちゃになりエラー対応に苦戦した。

### 3日目 (7月10日)
- SQLite3を使用したデータベースの導入
- SQL言語の基本機能を学習
- **所感**: SQL言語を全く知らなかったため、大半の時間を学習に費やした。

### 2日目 (7月9日)
- 簡単なCRUD機能を実装
- **所感**: 編集機能を実装した際、クリックできるのに編集できないという問題が発生。コードを見直し無事に解決できた。

### 1日目 (7月8日)
- Python Web開発の基本を習得するため、まずはFlaskで実装を開始（21日からはFastAPIへ移行予定）
- **所感**: HTMLの知識は講義で扱った基礎程度のため、今後コードが複雑化していくことに少し不安。

---

## 🛠 使用技術
- **Language**: Python
- **Framework**: Flask
- **Database**: SQLite3, SQLAlchemy (ORM)
- **Frontend**: HTML5, CSS3, Jinja2
